# Карта исходных материалов

Этот файл нужен как внутренняя навигация по исходным проектам. Он не копирует учебные задания, а показывает, из каких выполненных частей был собран отчет.

## Исходные папки

- `D01_Linux.ID_356272-1` - базовое администрирование Linux.
- `DO2_LinuxNetwork.ID_356275-2` - сетевое администрирование Linux.

## Основные подтверждающие файлы

- `D01_Linux.ID_356272-1/src/report.md` - исходный отчет по базовой установке и администрированию Ubuntu Server.
- `D01_Linux.ID_356272-1/src/img/` - скриншоты команд, конфигураций, логов и проверок.
- `DO2_LinuxNetwork.ID_356275-2/src/report.md` - исходный отчет по сетям, маршрутизации, firewall, DHCP и NAT.
- `DO2_LinuxNetwork.ID_356275-2/src/img/` - скриншоты сетевых конфигураций и проверок.

## Что вошло в общий отчет

### Из базового Linux-проекта

- установка Ubuntu Server 24.04 LTS без GUI;
- создание пользователя и настройка административных прав;
- hostname, timezone, loopback, DHCP, статический IP, gateway, DNS;
- обновление пакетов;
- синхронизация времени;
- работа в Vim, Nano и Mcedit;
- установка и настройка SSHD, изменение порта, проверка процесса и порта;
- top/htop для анализа процессов и ресурсов;
- fdisk/df/du/ncdu для анализа дисков и файловых систем;
- просмотр dmesg/syslog/auth.log и проверка события рестарта SSH;
- cron-задача, проверка выполнения и очистка расписания.

### Из Linux Network-проекта

- расчет сетей, масок, диапазонов и private/public-адресов;
- статическая маршрутизация между двумя виртуальными машинами;
- постоянные маршруты через netplan;
- измерение скорости через iperf3;
- iptables-фильтрация, порядок правил, ping/nmap-проверка;
- построение многоузловой сети с рабочими станциями и маршрутизаторами;
- IP forwarding и таблицы маршрутизации;
- traceroute/tracepath и tcpdump для анализа пути пакетов;
- ICMP-диагностика недоступной сети;
- DHCP-сервер, выдача адресов и привязка по MAC;
- NAT, SNAT, DNAT, Apache2 и telnet-проверки.

## Принцип переработки текста

В новом отчете намеренно не используются формулировки учебных заданий. Вместо этого каждая часть описана как практический инженерный опыт:

- какая область была изучена;
- какие действия выполнялись;
- какими инструментами проверялся результат;
- какой навык можно показать работодателю;
- какой вывод был сделан после практики.
